Setup Instructions

Phase 1: Physical Setup

Mount the GFCI power strip to your test bench using the hang holes. All test devices plug into this strip. Test the GFCI by pressing the TEST button—it should trip immediately.
Install the Shelly Pro 4PM in a DIN-rail enclosure near the bench. Wire it between the GFCI strip and 4 dedicated test outlets. Each channel controls one outlet. Follow Shelly's wiring guide for your electrical configuration (have an electrician verify if needed).
Connect the Shelly to your network via Ethernet (recommended) or Wi-Fi. Open the Shelly app or navigate to its IP address to configure. Note the IP—you'll need it for QuickRefurbz.
Set up the USB hub on the bench near your computer. Plug in the 36W power adapter. Connect the multimeter, logic analyzer, and label printer via USB.
Connect the Rigol DM3058E via the USB Device port on the rear panel. The instrument should appear as a serial device (e.g., /dev/ttyUSB0 on Linux/Mac).

Phase 2: Software Setup

Clone and build QuickRefurbz: 
git clone https://github.com/Quicklotz/QuickRefurbz.git
cd QuickRefurbz
npm install
npm run build
Initialize the database: 
node dist/enhanced-cli.js init
Scan for connected instruments: 
node dist/enhanced-cli.js scan
This should detect the Rigol DM3058E on its USB serial port.
Register the multimeter: 
node dist/enhanced-cli.js instrument add
Follow the interactive prompts. Select "SCPI Multimeter", enter the connection path from the scan output.
Verify communication: 
node dist/enhanced-cli.js measure <instrumentId> "*IDN?"
Should return the instrument's identity string (manufacturer, model, serial, firmware).
Install sigrok-cli (for the logic analyzer): 
# macOS
brew install sigrok-cli

# Ubuntu/Debian
sudo apt install sigrok-cli
Install Brother printer drivers from brother-usa.com/support/ql810w. The QL-810W works over USB or Wi-Fi.

Phase 3: Create a Test Station

Create a station via the QuickTestz API or web UI. Set:
  • Name: "Bench 1" (or whatever you call it)
  • Controller Type: SHELLY_GEN2_HTTP
  • Controller Base URL: http://<shelly-ip> (e.g., http://192.168.1.100)
  • Safety Flags: GFCI Present = true, Acknowledged By = your name
Add outlets to the station (one per Shelly channel):
  • Channel 0: "Outlet A", Max Amps: 15, On/Off: Yes, Metering: Yes
  • Channel 1: "Outlet B", etc.
Verify the station: 
node dist/enhanced-cli.js station list
node dist/enhanced-cli.js station health <stationId>

Phase 4: Run Your First Integrated Test

Check the overview dashboard: 
node dist/enhanced-cli.js overview
Verify your station, instruments, and any active runs are displayed.
Run an integrated test (hardware-diag + QuickTestz): 
node dist/enhanced-cli.js test start QLID000001 \
  -c VACUUM \
  --station <stationId>
This will: energize the outlet via Shelly, run the SCPI test plan, record all readings, compute a PASS/FAIL score, and de-energize.
Review results: 
node dist/enhanced-cli.js testrun list
node dist/enhanced-cli.js testrun readings <runId>
node dist/enhanced-cli.js test result QLID000001

                |  (36W Powered)    |

  Data Flow:
  1. CLI sends HTTP to Shelly -> energize outlet -> appliance powers on
  2. Shelly polls power (W/V/A) -> recorded as test_readings
  3. Rigol measures via SCPI -> recorded as hardware_test_step_results
  4. SCPI results also mapped -> additional test_readings
  5. Safety monitor watches for spikes -> auto-shutdown if threshold exceeded
  6. Test completes -> PASS/FAIL computed -> label printed
